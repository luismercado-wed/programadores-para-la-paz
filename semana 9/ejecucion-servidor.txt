¿Qué ocurre cuando ejecutas node server.js?
Al ejecutar el comando, Node.js compila e interpreta el archivo server.js en la memoria RAM, iniciando un proceso continuo que levanta el framework Express, asocia la aplicación al socket de red del puerto 3000 e ingresa en un bucle de eventos (Event Loop) listo para escuchar peticiones HTTP entrantes.

¿Qué mensaje aparece en la terminal?
Aparece el string configurado en el callback del método listen:
"Servidor ejecutándose en puerto 3000"

¿Qué sucede cuando visitas la ruta /estado?
El servidor intercepta la petición HTTP GET en el endpoint '/estado', procesa la instrucción y devuelve al cliente (navegador o Postman) una respuesta con código de estado 200 OK y un cuerpo en formato JSON estructurado de la siguiente manera:
{
  "estado": "Servidor funcionando",
  "servicio": "API comunitaria"
}
